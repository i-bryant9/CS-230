Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
Gaming room requested that Draw It or Lose It function on multiple platforms. The new application will be web based, while the original was android based. Games include multiple teams and players. Games, teams, and players must be uniquely named. There should be checks in place to allow users to verify the identifiers are unique. 
What did you do particularly well in developing this documentation?
In the earlier assignments i believe i executed the singleton design pattern particularly well as i understood this assignment most. compared to the memory management assignment where my work relied heavily on articles i read and the authors understandings. 
What about the process of working through a design document did you find helpful when developing the code?
The best practice in coding i believe is establishing requirements upfront so you can code in the requirements reducing any rework in the future and the software design document provides that framework.

i would have liked to have spent more time on the OS overviews and recommendations. If life would have allowed i would have went further in depth in my research so that i could have gained a better understanding of the OS and provided the best recommendations rather than the recommendations i formed from the few articles that i read. 
How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?
When interpreting the users needs it is important to provide the most valuable product. One requirement that i had to take into account was the uniqueness of the identifiers. Using the singleton pattern i was able to provide a feature that ensure there were no 2 like games, players, and teams.
